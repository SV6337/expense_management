# Expense Tracker (MERN)

This workspace contains a React frontend in `expense/` and an Express + MongoDB backend in `backend/`. It is set up for local development, Docker, and Kubernetes deployment.

## Folder Tree

```text
050/
├── README.md
├── docker-compose.yml
├── ec2-ubuntu-setup.yml
├── expense-key.pem
├── backend/
├── expense/
└── k8s/
```

## What Each File And Folder Does

### Root files

- `README.md` - Project overview, structure, and run instructions.
- `docker-compose.yml` - Starts the frontend, backend, and Redis containers together for local containerized development.
- `ec2-ubuntu-setup.yml` - Provisioning/setup notes for the EC2 host environment.
- `expense-key.pem` - SSH key file used for EC2 or deployment access; keep it private.

### `backend/`

```text
backend/
├── .env
├── .env.example
├── .gitignore
├── Dockerfile
├── package.json
├── package-lock.json
├── README.md
├── server.js
├── config/
│   ├── db.js
│   └── redis.js
├── controllers/
│   ├── authController.js
│   └── transactionsController.js
├── middleware/
│   └── auth.js
├── models/
│   ├── User.js
│   └── Transaction.js
├── routes/
│   ├── auth.js
│   └── transactions.js
└── scripts/
	└── cleanupLegacyTransactions.js
```

- `.env` - Local backend environment variables such as `MONGO_URI`, `PORT`, `JWT_SECRET`, and optional `REDIS_URL`.
- `.env.example` - Template showing the expected backend environment variables.
- `.gitignore` - Ignores local build artifacts, dependencies, and environment files.
- `Dockerfile` - Builds the backend container image.
- `package.json` - Backend scripts and runtime dependencies.
- `package-lock.json` - Locked dependency versions for the backend.
- `README.md` - Backend-specific notes and run instructions.
- `server.js` - Express entry point; wires middleware, routes, and database startup.
- `config/db.js` - MongoDB connection helper.
- `config/redis.js` - Redis helper used for transaction caching.
- `controllers/authController.js` - Signup, login, profile update, and password verification logic.
- `controllers/transactionsController.js` - Create, read, and delete transaction logic.
- `middleware/auth.js` - JWT protection middleware for private routes.
- `models/User.js` - MongoDB user schema and password hashing/comparison.
- `models/Transaction.js` - MongoDB transaction schema.
- `routes/auth.js` - Auth API routes.
- `routes/transactions.js` - Transaction API routes.
- `scripts/cleanupLegacyTransactions.js` - Maintenance script for removing old transactions without ownership.

## Quick Start

1. Start MongoDB (local or Atlas) and set `MONGO_URI` if you are not using the default local database.
2. In one terminal, run the backend:

```powershell
cd backend
npm install
npm run dev
```

3. In another terminal, run the frontend:

```powershell
cd expense
npm install
npm start
```

The frontend uses `REACT_APP_API_BASE_URL` to point to the backend API, and in production it should target your EC2 NodePort base URL.
